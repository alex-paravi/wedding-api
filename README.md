# Wedding Guest List & Table API (Laravel)

Простой бэкенд-сервис (REST API) для управления списками гостей, их рассадкой и подтверждением присутствия на свадьбе. Проект разработан в рамках перехода на профессиональный бэкенд-уровень с упором на оптимизацию запросов и чистую архитектуру данных.

## 🚀 Технологический стек

- **Язык:** PHP 8.x
- **Фреймворк:** Laravel 11.x (API-роутинг)
- **База данных:** SQLite / MySQL (Eloquent ORM)
- **Архитектурные паттерны:** REST API, API Resources (трансформеры данных), Eager Loading (борьба с N+1)

## 🛠 Архитектурные особенности

- **Оптимизация производительности:** В `GuestResource` реализована условная загрузка отношений через метод `$this->whenLoaded()`. Это защищает базу данных от проблемы N+1 запросов при выводе списков. Данные связанных сущностей (столы, создатель) подтягиваются только при явном вызове `with()`.
- **Связи данных:** Настроены Eloquent-отношения между пользователем-организатором (`User`), гостями (`Guest`) и столами (`Table`). Вся структура синхронизирована с базой данных SQLite.

## 📋 Реализованный функционал (CRUD)

### Управление гостями (`/api/guests`)

- `GET /api/guests` — Получение списка гостей (с ленивой загрузкой связанных столов).
- `GET /api/guests/{id}` — Просмотр карточки конкретного гостя (обработка 404 Not Found).
- `POST /api/guests` — Добавление нового гостя в список.
- `PUT /api/guests/{id}` — Полное обновление данных гостя (включая привязку к столу `table_id` и статус).
- `DELETE /api/guests/{id}` — Удаление гостя из базы данных (возвращает 204 No Content).

### Управление столами (`/api/tables`) — _В процессе_

- Маршрутизация на базе единого `Route::apiResource`.

## 💻 Как запустить проект локально

1. Клонировать репозиторий:
   git clone https://github.com/alex-paravi/wedding-api.git
   cd wedding-api

2. Установить зависимости Composer:
   composer install

3. Настроить файл окружения (создать .env из примера):
   cp .env.example .env

4. Сгенерировать ключ приложения:
   php artisan key:generate

5. Запустить миграции для создания таблиц в БД:
   php artisan migrate

6. Запустить локальный сервер:
   php artisan serve

## 🎯 Тестирование API (Postman)

В корень репозитория приложен файл интерактивной коллекции: `Wedding Guest List API.postman_collection.json`.

1. Импортируйте этот файл в Postman (кнопка _Import_ в левом верхнем углу).
2. Запустите локальный сервер Laravel (`php artisan serve`).
3. В коллекции настроены все методы и заголовок `Accept: application/json`. Можно сразу отправлять запросы и проверять ответы бэкенда.
