# Wedding Guest List API (Laravel)

Простой бэкенд-сервис (REST API) для управления списком гостей и подтверждения их присутствия на свадьбе. Проект разработан в рамках практического освоения архитектуры веб-приложений и фреймворка Laravel.

## Технологический стек

- **Язык:** PHP 8.x
- **Фреймворк:** Laravel 11.x (API-роутинг)
- **База данных:** SQLite / MySQL (Eloquent ORM)
- **Архитектура:** REST API, базовый CRUD-цикл

## Реализованный функционал (CRUD)

В рамках проекта полностью реализовано ядро управления сущностью Guest:

- GET /api/guests — Получение полного списка приглашенных гостей.
- GET /api/guests/{id} — Просмотр карточки конкретного гостя (с обработкой ошибки 404 Not Found).
- POST /api/guests — Добавление нового человека в список.
- PATCH /api/guests/{id} — Частичное обновление данных (изменение статуса подтверждения присутствия is_confirmed).
- DELETE /api/guests/{id} — Удаление гостя из базы данных (возвращает статус 204 No Content при успехе).

## Как запустить проект локально

1. Клонировать репозиторий:
   git clone https://github.com/alex-paravi/wedding-api.git
   cd wedding-api

2. Установить зависимости Composer:
   composer install

3. Настроить файл окружения (создать .env из примера):
   cp .env.example .env

4. Сгенерировать ключ приложения:
   php artisan key:generate

5. Запустить миграции для создания таблиц в БД:
   php artisan migrate

6. Запустить локальный сервер:
   php artisan serve

## Презентация и тестирование API (Postman)

Для удобства проверки функционала без фронтенда, в корень репозитория приложен файл интерактивной коллекции для Postman: Wedding Guest List API.postman_collection.json.

1. Импортируйте этот файл в Postman (кнопка Import в левом верхнем углу программы).
2. Запустите локальный сервер Laravel (php artisan serve).
3. В коллекции уже настроены все 5 методов и необходимый заголовок Accept: application/json для корректного получения ответов. Вы можете сразу отправлять запросы и тестировать бэкенд.
